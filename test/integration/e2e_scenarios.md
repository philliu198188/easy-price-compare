## 场景 1：搜索商品 → 查看结果 → 点入详情 → 返回

**核心用户路径：搜索 → 浏览 → 决策 → 返回**

### 步骤

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1.1 | 启动 App，看到首页搜索框 | 首页渲染：顶部"易比价"标题 + 搜索框 + 搜索历史/热门搜索 Tab |
| 1.2 | 在搜索框输入"iPhone" | 搜索框内容显示"iPhone"，触发实时建议 |
| 1.3 | 从建议列表中选择"iPhone 15" | 导航到 `/results?q=iPhone%2015` |
| 1.4 | 搜索结果页加载完成 | 显示商品卡片网格（骨架屏 → 商品列表） |
| 1.5 | 点击第一个商品卡片 | 导航到 `/detail/{productId}` |
| 1.6 | 商品详情页展示 | 显示商品图片区域、价格信息、全平台比价列表 |
| 1.7 | 点击返回按钮 (←) | 返回到搜索结果页，保持滚动位置和结果 |
| 1.8 | 再次点击返回按钮 | 返回到首页 |

### 验证点

- [ ] 搜索词在 URL 中正确编码传递
- [ ] 搜索结果从骨架屏平滑过渡到实际内容
- [ ] 详情页正确接收并显示 productId
- [ ] 返回操作保持页面栈正确状态
- [ ] GoRouter 导航无异常

## 测试数据准备

测试应使用 mock 数据，避免依赖真实 API：

- **Mock API 服务**：使用 `http.Client` mock 拦截请求
- **Mock SharedPreferences**：使用 `SharedPreferences.setMockInitialValues`
- **Mock 商品数据**：预定义 10-20 条商品 JSON
- **延迟模拟**：使用 `Future.delayed` 模拟网络延迟
